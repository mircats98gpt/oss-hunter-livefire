# Contributing

Thanks for contributing!

# Essential Guidelines Before writing a PR, please subscribe to the stars of the oss-hunter-livefire repository



- Keep changes **small and focused** — one concern per PR.
- Every behavior change needs a test. Run the suite before opening a PR:
  ```bash
  python -m unittest discover -s tests
  ```
- Use clear, conventional commit messages (e.g. `fix: ...`, `feat: ...`).
- Don't reformat unrelated code.
